## Cover page (outer)

### Required elements (in order, centered)

**1. University Header** (13pt, bold, uppercase):

```
MINISTRY OF EDUCATION AND TRAINING
CAN THO UNIVERSITY
COLLEGE OF INFORMATION AND COMMUNICATION TECHNOLOGY
DEPARTMENT OF SOFTWARE ENGINEERING
```

**2. University Logo** (3cm diameter, centered)

**3. Thesis Type** (14pt, bold, uppercase):

```
GRADUATION THESIS
BACHELOR OF ENGINEERING IN
INFORMATION TECHNOLOGY
(High-Quality Program)
```

**4. Thesis Title** (18pt, bold, uppercase, max 2-3 lines)

**5. Student Information** (13pt, two-column layout):

```
Student:          [Full Name]
Student ID:       [B1234567]
Class:            [Class Name]
Advisor:          [Dr./Assoc. Prof. Name]
```

**6. Location and Date** (13pt):

```
Can Tho, Month/Year
```

**Border**: 2pt blue border (#003399 — CTU Blue) with 1.5cm inset

## Inner cover page

Same as outer cover but with:
- 3pt border
- 1cm inset
- Slightly larger logo (3.5cm)

## Evaluation page

**Title**: "EVALUATION OF ADVISOR" (centered, bold, uppercase)

**Content**:
- 20 blank lines for handwritten comments
- Signature section:

```
Advisor: _______________

[Signature space]

Dr./Assoc. Prof. [Name]
```

## Acknowledgements

**Title**: "ACKNOWLEDGEMENTS" (centered, bold, uppercase)

**Content** (1-2 pages):
- Thank advisor
- Thank university/department
- Thank family and friends
- Personal signature and date

**Format**:
- First-line indent: 1cm
- Justified text
- Signature: right-aligned

## Abstract

**Title**: "ABSTRACT" (centered, bold, uppercase)

| Requirement | Value |
|-------------|-------|
| **Length** | 200-350 words (strictly enforced) |
| **Keywords** | 3-5 keywords |
| **Keyword format** | `Keywords: keyword1, keyword2, keyword3` |
| **Keyword position** | After abstract, separated by 1cm space |

### Abstract structure

1. Background and motivation (2-3 sentences)
2. Objectives and methodology (3-4 sentences)
3. Main results and findings (3-4 sentences)
4. Conclusions and implications (2-3 sentences)

## List of figures

Auto-generated. Each entry is numbered as `Figure <chapter>.<number>`.

Example:

```
Figure 1.1. System Architecture Diagram .................. 12
Figure 2.1. Database Schema .............................. 25
Figure 2.2. User Interface Mockup ......................... 28
```

## List of tables

Auto-generated. Each entry is numbered as `Table <chapter>.<number>`.

Example:

```
Table 2.1. Performance Comparison Results ................ 30
Table 3.1. Test Case Summary ............................. 42
```

## List of abbreviations

**Title**: "LIST OF ABBREVIATIONS" (centered, bold, uppercase)

**Format**: Two-column table

| Abbreviation | Description |
|--------------|-------------|
| API | Application Programming Interface |
| CTU | Can Tho University |
| HTTP | Hypertext Transfer Protocol |

**Sorting**: Alphabetical order

Edit in `info.typ` under `abbreviations`:

```typst
abbreviations: (
  ("API", "Application Programming Interface"),
  ("CTU", "Can Tho University"),
  ("HTTP", "Hypertext Transfer Protocol"),
)
```

## Page numbering for front matter

- Lowercase Roman numerals (i, ii, iii...)
- Bottom center position
- Starts from Table of Contents (page i)
- Cover pages not numbered
